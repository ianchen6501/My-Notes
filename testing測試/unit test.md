## 定義
針對 function / method 驗證輸入、輸出的測試方法。
分為 TDD(Test-Deriven Development) / BDD(Behavior-Deriven Development) 兩種

## TDD
先寫測試程式，再實作功能。  
TDD 三定律：  
(1) 沒有測試之前不要寫任何功能代碼  
(2) 只編寫恰好能夠體現一個失敗情況的測試代碼  
(3) 只編寫恰好能通過測試的功能代碼

## BDD
使用 describe() 和 it()  
需求為導向的設計  
語意化
